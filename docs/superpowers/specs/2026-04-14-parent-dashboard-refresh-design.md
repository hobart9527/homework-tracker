# 家长端 Dashboard 改版设计

**日期：** 2026-04-14
**范围：** 家长首页、提醒链路、新建作业页、设置页、作业管理页默认带入体验

## 目标

这次改版聚焦家长端高频使用体验，目标是让家长进入首页后先处理“今天该盯什么”，再回看“本月整体怎么样”，并把提醒孩子完成作业这件事变成单条可执行动作。

本次设计覆盖以下问题：

- 家长首页当前月历占用空间大，但信息密度偏低，缺少月切换、悬浮详情和统计概览
- 当天进度和待办优先级不够高，无法快速执行跟进动作
- 当天作业没有单条提醒入口
- 新建作业页快捷类型位置靠后，输入路径割裂
- 从孩子筛选视角进入新建作业时，不会默认带入孩子，容易误分配

## 设计原则

- 当天优先：第一屏优先服务“今天还差什么、现在该提醒谁”
- 月度清晰：月历从大块文本改成轻量可扫视的信息单元
- 行动闭环：提醒不是静态标签，而是能触发并反馈状态的操作
- 保留熟悉路径：恢复原来更顺手的快捷类型优先录入体验
- 为后续扩展留接口：先落地单条提醒，同时预留规则化升级提醒能力

## 信息架构

家长首页改成以下顺序：

1. 当天进度区
2. 本月打卡日历
3. 本月薄弱类型
4. 作业打卡热点时间段分析

其中“当天进度区”是第一屏核心区域，负责承接家长进入页面后的第一反应；“本月打卡日历”和统计信息承接回顾和发现问题；“薄弱类型”和“热点时间段”作为跟进和策略支持信息。

## 首页布局

### 1. 当天进度区

当天进度区展示当前选中孩子在选中日期下的详细状态。默认日期为今天，默认孩子为当前排序优先级最高的孩子。

区域结构：

- 头部：日期、孩子头像、孩子名称、当天顶部提示语
- 四个指标卡：完成率、当天积分、待完成数、逾期数
- 当天作业清单：每条作业展示标题、类型图标、截止时间、证明要求、当前状态、积分结果
- 每条作业右侧增加单条操作按钮：`提醒孩子`

交互规则：

- 点击月历某一天后，此区域切换到该日期的数据
- 切换孩子后，此区域显示该孩子在当前选中日期的数据
- 若该日没有任务，展示空状态和返回今天的快捷入口
- 若该作业已完成，则隐藏提醒动作
- 若该作业已有未结束的提醒流程，按钮改为状态按钮，显示下次升级时间或已升级状态

### 2. 本月打卡日历

日历从当前的文字型卡片改成轻量比例环形式单元，单元目标是“更省空间，但仍一眼看懂当天完成情况”。

单元内容：

- 日期数字
- 一个比例环形图，表示当日完成比例
- 环形颜色分段区分三类状态：
  - 按时完成
  - 补做完成
  - 未完成

交互规则：

- 支持按月切换，提供上个月 / 下个月导航
- 悬浮某一天时显示浮窗，内容包括：
  - 日期
  - 总作业数
  - 已完成数
  - 补做数
  - 未完成作业标题列表
- 点击日期后，联动更新上方当天进度区
- 当前选中日期有明确高亮态

### 3. 月度统计

月历头部增加当月统计概览，采用 4 个轻量统计卡：

- 本月完成率
- 本月按时完成率
- 本月总积分
- 本月补做天数

这些统计依附于当前选中孩子和当前浏览月份，不做“全局混合看板”，避免解释成本上升。

### 4. 本月薄弱类型

保留“本月薄弱类型”，但位置后移到日历之后。展示逻辑仍然按完成率从低到高排序，优先显示最需要关注的类型。

可补充字段：

- 类型名
- 完成率
- 完成数 / 分配数
- 环比提示预留位（本次不实现真实环比，仅保留布局可能性）

### 5. 作业打卡热点时间段分析

新增“热点时间段分析”，用于帮助家长判断孩子通常在什么时间完成作业，辅助安排提醒节奏。

首版分析口径：

- 统计当前孩子、当前月内所有 check-in 的完成时间
- 按小时段归类
- 输出一个热力条形或简化热力带
- 展示峰值时间段，例如“19:00-20:00 最常完成”

如果当月样本不足，则显示“数据不足，继续打卡后会生成分析”。

## 提醒链路设计

### 目标

让家长能对单条未完成作业发起一次即时提醒，并在孩子 2 小时后仍未完成时自动升级为电话提醒。

### 用户流程

1. 家长在当天作业清单里点击某条作业的 `提醒孩子`
2. 系统立即发送首次短信提醒
3. 系统创建 reminder 记录，状态为“短信已发，等待升级检查”
4. 若 2 小时内作业完成，则 reminder 状态更新为“已完成，停止升级”
5. 若 2 小时后仍未完成，则自动发起语音电话提醒
6. reminder 状态更新为“已升级电话”

### 页面反馈

当天作业卡片上需要展示提醒状态：

- 未提醒：显示 `提醒孩子`
- 已发短信待升级：显示 `已短信提醒 · 2小时后未完成将电话提醒`
- 已取消升级：显示 `提醒已结束`
- 已升级电话：显示 `已电话提醒`
- 发送失败：显示 `提醒失败，可重试`

### 渠道策略

- 首次提醒渠道：短信
- 升级提醒渠道：语音电话
- 本次不做渠道切换 UI，先按固定策略实现

### 实现边界

- 本次先支持“单条作业触发”
- 不支持批量提醒
- 不支持家长自定义升级时长
- 不支持多次循环升级
- 不支持复杂渠道编排

## 数据与系统设计

### Dashboard 数据扩展

现有 `buildParentDashboard` 需要按月份输出更多聚合结果：

- 月度统计概览
- 指定月份的日历天数据
- 每天的 hover 摘要信息
- 热点时间段统计

建议新增以下数据结构：

- `ParentMonthlyStats`
- `ParentCalendarDayTooltip`
- `ParentReminderState`
- `ParentCheckInHeatmapBucket`

### Reminder 数据模型

建议新增 reminder 记录表，字段至少包括：

- `id`
- `parent_id`
- `child_id`
- `homework_id`
- `target_date`
- `status`
- `initial_channel`
- `escalation_channel`
- `initial_sent_at`
- `escalate_after`
- `escalated_at`
- `resolved_at`
- `failure_reason`
- `created_at`

状态建议：

- `pending_initial`
- `sent_sms`
- `resolved_completed`
- `escalated_call`
- `failed`

### Reminder 执行方式

本次采用混合版策略：

- 前端点击单条提醒时，调用新的提醒 API
- API 立即发送短信，并创建 reminder 记录
- 后台定时任务扫描 `sent_sms` 且超过 `escalate_after` 的记录
- 若关联作业仍未完成，则触发语音电话，并更新状态
- 若已完成，则标记为 `resolved_completed`

这样本次能快速上线单条提醒，也给后续规则化提醒保留了标准状态机。

## 新建作业与设置页调整

### 新建作业页

恢复成“快捷类型优先”的填写顺序：

1. 快捷类型
2. 作业标题
3. 描述
4. 其他规则和奖励字段

交互规则：

- 选择快捷类型后，自动带入标题建议、图标和默认积分
- 若用户已手改标题，再切换快捷类型时不覆盖手动标题
- 新建作业页不再承担“新建快捷类型”的主要入口

### 设置页

将“快捷类型管理”提升为设置页第一项，优先于提醒设置。

设置页顺序改为：

1. 快捷类型管理
2. 提醒设置
3. 账户

快捷类型管理区负责：

- 查看已有自定义类型
- 新建类型
- 编辑图标与默认积分
- 删除类型

## 作业管理页体验修复

从作业管理页里某个孩子的清单视角进入 `新建作业` 时，默认带入该孩子 ID。

行为规则：

- 若当前处于“全部孩子”，点击新建则维持默认新建逻辑
- 若当前处于某个孩子筛选态，点击新建则通过 query 参数带入 `childId`
- 新建页初始表单直接选中该孩子
- 如果是复制已有作业，则优先使用复制来源的原始孩子；若用户从单孩子视角新建且不是复制场景，则使用 query 参数孩子

## 测试策略

### Dashboard

- 月切换后显示对应月份数据
- 点击日历日期会联动更新当天进度区
- hover 浮窗展示当天摘要
- 月度统计计算正确
- 热点时间段统计在有样本和无样本两种情况下都能正常显示

### Reminder

- 点击单条提醒后触发短信并创建 reminder 记录
- 2 小时内作业完成时，升级提醒被取消
- 2 小时后作业仍未完成时，触发语音电话
- 失败状态在 UI 上可见并可重试

### Homework Form / Settings / Homework List

- 快捷类型位置前移后仍能正确联动标题与积分
- 设置页可管理快捷类型
- 单孩子视角点击新建时默认带入孩子
- 复制作业时不会错误覆盖原始孩子分配逻辑
